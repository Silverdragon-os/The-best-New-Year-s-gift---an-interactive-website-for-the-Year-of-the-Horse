# Contributing to Stellar Steed 🐎

First off, thank you for considering contributing to the **Stellar Steed** project! It's people like you who make the open-source community such an amazing place to learn, inspire, and create.

当你在为“星空灵驹”贡献力量时，请遵循以下指南：

---

## 🛠️ How Can I Contribute? / 如何贡献？

### 1. Reporting Bugs / 反馈 Bug
如果您在交互中发现手势识别不准、模型消失或性能卡顿：
* 请先检查 [Issues](https://github.com/你的用户名/stellar-steed/issues) 页面，看是否有人已经提出。
* 如果没有，请提交一个新的 Issue，并详细描述您的设备环境（如：Chrome 版本、手机型号）。

### 2. Suggesting Enhancements / 提出改进
我们非常欢迎关于视觉效果或交互方式的创意：
* 想要更多的生肖粒子模型？
* 想要更酷炫的爆炸音效？
* 请在 Issues 中开启一个功能请求 (Feature Request)。

### 3. Pull Requests / 提交代码
如果您想直接修改代码：
1. **Fork** 本仓库并创建您的开发分支 (Branch)。
2. **Commit** 您的修改。请确保代码简洁并添加了必要的英文注释。
3. **Sign-off**: 为了保证代码的版权合规，请在提交时勾选 **"Sign off on this commit"** (或在命令行使用 `git commit -s`)。
4. **Push** 分支并向主仓库提交一个 **Pull Request**。

---

## 🎨 Development Guidelines / 开发规范

* **Code Style**: 保持原生 JavaScript (ES Modules) 风格，尽量避免引入厚重的第三方库，以维持项目的轻量性。
* **Visuals**: 如果修改粒子算法，请务必在移动端进行流畅度测试 (目标 60FPS)。
* **Language**: 所有的 UI 提示请保持英文，注释建议使用英文，以便全球开发者理解。

---

## ⚖️ Legal / 法律声明

当你向本项目提交代码时，即表示你同意：
1. 你的贡献将遵循项目的 **MIT License**。
2. 你签署的 **DCO (Developer Certificate of Origin)** 声明你的代码是原创的或拥有合法的授权。

---
